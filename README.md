## Overview

This portfolio demonstrates hands-on GRC (Governance, Risk, and Compliance) skills through four projects that mirror real deliverables produced by entry-level GRC Analysts and IT Auditors. All projects assess a fictional organization — **TechBridge Solutions** — providing a consistent, realistic scenario across every deliverable.

## Project Details

### Project 1 — Simple Risk Register (`01_Risk_Register.xlsx`)
A structured Excel workbook identifying 10 organizational risks scored by Likelihood × Impact. Includes three tabs:
- **Risk Register** — Full risk inventory with descriptions, ratings, mitigations, and owners
- **Risk Summary Dashboard** — Count and percentage of risks by rating (Critical/High/Medium/Low)
- **Remediation Tracker** — Action items, owners, target dates, and status

**Skills:** Risk identification, L×I scoring, heat map logic, Excel dashboards, remediation tracking

---

### Project 2 — Policy Review (`02_Policy_Review.docx`)
A formal review of TechBridge Solutions' Acceptable Use Policy identifying 6 gaps against NIST CSF and ISO 27001, each rated by severity (High/Medium/Low) and mapped to specific control references. Includes actionable recommendations and a defined next-steps section.

**Skills:** Policy analysis, framework mapping (NIST CSF, ISO 27001), written communication, gap identification

---

### Project 3 — Mini Gap Assessment (`03_Mini_Gap_Assessment.docx`)
A full NIST CSF v1.1 gap assessment covering 17 controls across all five Functions — Identify, Protect, Detect, Respond, and Recover. Each control is assessed as Implemented / Partial / Not Implemented, with current state documentation and remediation guidance. Includes a prioritized 180-day remediation roadmap.

**Skills:** NIST CSF v1.1, control-level assessment, risk prioritization, governance documentation

---

### Project 4 — Audit-Style Finding Report (`04_Audit_Finding_Report.docx`)
Four formal audit findings documented using the standard **5C audit format**: Condition, Criteria, Cause, Effect, Recommendation. Findings cover MFA, Incident Response, Backup Testing, and Patch Management — four of the most common IT audit areas. Includes a management response table with owners and target dates.

**Skills:** Audit writing, 5C finding format, root cause analysis, management response documentation, CIS Controls, NIST SP 800-61
